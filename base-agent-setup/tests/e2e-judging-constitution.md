# E2E Judging Constitution

The rubric independent judges score against during the iterative improvement loop. Each artifact (brain-doc, system prompt, discovery prompt) gets scored 0–10 by three independent judges per round. Each judge sees ONLY: the artifact, this constitution, the four reference example agent prompts in `templates/example-agents/`, and the corresponding generator prompt (e.g. `prompts/synthesize-brain-doc.md`). Judges DO NOT see other judges' scores or any prior round's artifacts/scores.

A round passes when all three artifacts score ≥ 9 from all three judges. If any score is < 9, the highest-impact issue across all judge feedback drives the next round's targeted fix.

---

## Universal cross-cutting criteria (apply to all three artifacts)

### C1. Vendor-name hygiene (LOAD-BEARING)

Run `grep -iE 'ultravox|telnyx|supabase|n8n|resend|railway|firecrawl|anthropic|\bclaude\b|\bopus\b|\bhaiku\b'` on the artifact. **Any match drops the score for that artifact to MAX 4 regardless of other quality.** Vendor names must NEVER appear in any artifact reaching the customer.

Exception: the artifact may mention "ChatGPT" — that's the platform the customer uses, not our backend.

### C2. Examples-not-scripts

Prescriptive call scripts ("when caller says X, you say Y") are a hard fail. Behavioural patterns and example dialogue snippets are good. The agent should be told what good behaviour looks like, given exemplar phrasings, and trusted to think on its feet — not given a rigid decision tree to follow word-for-word.

**Heuristic:** if the artifact contains step-by-step "first do A, then do B, then ask C" sequences for a *complete call flow*, it's scripted (bad). If it contains scenario-grouped guidance with example phrasings ("when handling complaints, the tone is X — sample phrasings: '...'"), it's example-driven (good).

Score impact: scripts present anywhere → MAX 6. Pure example-driven → up to 10.

### C3. Reference-prompt parity

Compare against the four reference prompts at `templates/example-agents/*.prompt.md` (Steve, Hannah, Jack, Emma). The new artifact should be in the same depth, structure, and behavioural-richness ballpark. Specifically:

- **Identity & role** layer present and warm
- **Caller scenario coverage** — the prompt anticipates the kinds of calls the business actually receives, with patterns for each
- **Transfer/escalation logic** — clear, business-specific, not generic
- **Brand voice** — texture (not just adjectives — actual phrasings, hedges, signature moves)
- **Edge-case handling** — rude callers, broken English, ambiguous requests, common confusions specific to the vertical
- **Operational detail** — hours, pronunciation guides, hold/transcribe etiquette

Score impact: artifact at ≥ 80% of reference depth = up to 10. At 50–79% = max 7. Below 50% = max 5.

### C4. Bespoke vs generic

The artifact must clearly reference *this specific customer*'s details — their actual services, their actual locations, their actual brand specifics. Generic-receptionist content that could apply to any business of the same vertical = max 6.

### C5. Internal consistency

No contradictions between sections (e.g. brain-doc says "no emergency line" but system prompt routes emergencies to a number). No phrasing collisions. No duplicated content.

## Per-artifact criteria

### Brain-doc (`brain-doc.md`)

Generated by `prompts/synthesize-brain-doc.md` at Stage 3.

**Specific scoring criteria:**

- **B1. Required headings present.** All of: `## Identity`, `## Services`, `## Hours`, `## Locations & Service Area`, `## Staff`, `## Contact`, `## Policies & Pricing`, `## Tone & Voice`, `## Notable from Meeting`. Missing heading = max 6.
- **B2. Source flagging.** Every fact tagged with `[confirmed: site + meeting]`, `[from site only]`, `[from meeting only]`, or `[inferred]`. Unflagged facts = max 7. (For the no-meeting test run, "[from site only]" should dominate.)
- **B3. Size sanity.** 3–8 KB target. <1.5 KB = under-fed, max 5. >12 KB = padded, max 7.
- **B4. No fact-borrowing from examples.** Hard rule from the synthesis prompt: phone numbers, addresses, staff names, prices, transfer numbers from the reference example-agents must NOT appear in this brain-doc. Any leak = MAX 4.
- **B5. Tone & Voice section is texture-rich.** "Warm and professional" alone is empty. Should reference specific tone markers from the customer's site copy.

### Rough system prompt (`system-prompt.md`)

Generated by `prompts/assemble-rough-system-prompt.md` at Stage 4.

**Specific scoring criteria:**

- **S1. Four-section delimiters present and ordered.** Exactly: `=== UNIVERSAL_RULES ===`, `=== AGENT_IDENTITY ===`, `=== BRAIN_DOC ===`, `=== MINIMAL_TOOL_NOTE ===`. Missing/reordered = MAX 4.
- **S2. Universal rules verbatim.** The block under `=== UNIVERSAL_RULES ===` matches `templates/universal-rules.md` byte-for-byte (no truncation, no rephrasing). Diff = MAX 5.
- **S3. AGENT_IDENTITY substitution clean.** No literal `{customer_name}` or `{agent_first_name}` placeholder remains. Done = neutral; not done = MAX 3.
- **S4. Brain-doc content elevated, not just pasted.** The enrichment pass per `assemble-rough-system-prompt.md` should expand underdeveloped sections of the brain-doc using behavioural patterns from the example-agents (caller scenarios, hold messages, transfer phrasings, etc.) WITHOUT borrowing facts. The final brain-doc section should be RICHER than what Stage 3 originally produced. If the section is identical to Stage 3 output = max 7.
- **S5. Size 10–25 KB.** Per the spec.
- **S6. MINIMAL_TOOL_NOTE present and clear.** The agent is correctly told it has no action tools and knows how to gracefully degrade.
- **S7. Voice authenticity.** Reading the prompt aloud, does it sound like a real, vertical-appropriate receptionist with this business's specific voice? Or generic? Generic = max 6.

### Discovery prompt (`discovery-prompt.md` + optionally `customer-context.md`)

Generated by `prompts/generate-discovery-prompt.md` at Stage 10.

**Specific scoring criteria:**

- **D1. Tailored opening question.** Anchored in something specific from the website (or meeting if present). Generic opener ("tell me about your business") = MAX 4.
- **D2. Methodology embedded or attached.** The full methodology body from `reference-docs/discovery-methodology.md` is reachable by the customer's ChatGPT. Either inline (single-file mode) or via attached `customer-context.md` (two-file mode). Truncation = MAX 5.
- **D3. Known-vs-unknown map present.** The prompt explicitly tells ChatGPT what's already known about the business (so it doesn't re-ask) and what still needs the customer's input.
- **D4. Brief output schema spelled out.** Sections A–F with the format the customer should produce when emailing back.
- **D5. Posture rules present.** Optimistic-realistic; integration optimism; meeting-first scope inference; graceful hard-asks.
- **D6. Two-file-mode formatting clean (if applicable).** If `customer-context.md` is generated, the cover-email instructions tell the customer clearly to paste the prompt and drag-drop the context file. No ambiguity.
- **D7. Size — practical.** `discovery-prompt.md` ≤ ~10 KB if two-file mode; ≤ ~25 KB if single-file. Methodology body alone exceeds 10 KB → two-file mode is mandatory.
- **D8. Customer-readiness.** Read it as if you were the customer. Would you actually engage with it for 30 minutes? Or would you bounce after the first message? Bounce-likely = max 6.

## Scoring rules

- Whole numbers only. 0–10.
- A score below 9 must include at least one **specific, line-citable issue** (e.g. "system-prompt.md line 412 has a 5-step script for handling complaints"). Vague critiques don't count toward dropping the score.
- A score of 10 means: nothing to improve. The judge would ship this to a paying customer as-is. Don't give 10 lightly.
- A score of 0 means: actively harmful. Don't give 0 unless something is dangerous.

## Aggregation rule

Per artifact, take the **minimum** of the three judges' scores (not average). One judge spotting a real issue is reason to fix.

A round passes only when all three artifacts have minimum-score ≥ 9.

## Anti-cheat protocol

Each judge subagent:
- Receives ONLY the artifact + this constitution + the four reference example-agent prompts + the corresponding generator prompt.
- Receives a fresh subagent context with no memory of previous rounds.
- Must NOT read the conversation history or any prior judge's report.
- Must cite specific line numbers or quoted text for every score below 9.
- Must score independently — no consensus-seeking, no aggregation in the judge's own head.

If a judge cannot find specific cited evidence for a deduction, the deduction is removed and the score floor on that criterion is 9.
